fetch_listing_age_days() — cs/api.py
New helper — returns how many days ago a symbol first traded on Binance
First daily candle timestamp as listing proxyNew
Calls /api/v3/klines with interval=1d, limit=1, and startTime=1483228800000 (Jan 1 2017). The open_time of the returned candle is the first day the symbol ever traded. Age in days = (now_ms − first_open_ms) / 86_400_000. Returns None on any error so the caller can skip gracefully.
One call, minimal payload — 1 candle, daily intervalLogic
Using interval=1d and limit=1 means Binance returns the smallest possible response. The call resolves in under 200 ms and uses negligible API weight. It is only made per coin when the new listing filter is enabled — zero overhead when the filter is off.
🔍
New Listing Gate — cs/scanner.py
Runs before fetch_klines() — no wasted candle fetches on filtered-out coins
Gate fires before fetch_klines() and analyse()Logic
If CFG["new_listing_filter"] is true, fetch_listing_age_days(sym) is called first. If the age is outside [new_listing_min_days, new_listing_max_days] or returns None, the coin is skipped with continue — no kline fetch, no analysis, no alert check. This keeps the scan fast even with the filter enabled.
Filter is off by default — no behaviour change for existing usersImproved
CFG["new_listing_filter"] defaults to False. The gate code is inside an if CFG.get("new_listing_filter"): block, so it is never reached unless explicitly enabled. Settings from v2.7.x and earlier load without migration.

Config UI — cs/main_window.py
Three new controls in Config → Scan Filters
New listing filter checkbox + two spinnersNew
Added to the SCAN FILTERS group in the Config tab: a checkbox to enable/disable the filter, a Listed min spinner (1–30 days), and a Listed max spinner (2–90 days). All three persist across restarts via QSettings keys newListingFilter, newListingMinDays, newListingMaxDays. Apply Settings takes effect on the next scan.
