Crypto Scalper Scanner — Changelog v3.x

v3.3.0 Apr 1, 2026

Surge Detector — hardened gates, cooldown tuning, UI fixes

  • REMOVED surge_min_price_pct — Binance returns 0.0% 24h change for some coins even during big intraday moves (NOM +26% showed 0.0% 24h). Gate was unreliable regardless of threshold.
  • CHANGED surge_max_price_pct 30% → 100% — STO +75% on 24h exceeded old 30% gate, excluded before kline check despite 7× 5m volume at 16:57 UTC. Raised to 100% so only coins that literally doubled+ on 24h are skipped.
  • CHANGED surge_cooldown_mins 60 → 20 min — allows re-alerts during sustained breakouts (ONT fired at 03:08, 60-min cooldown blocked 09:36, 10:08, 10:22 spikes)
  • FIX Alert tab no longer hijacks Settings sub-tab on new alert
  • FIX Alert list now unlimited — removed 20-row UI cap, 50-item in-memory cap, 50-item JSON save cap. List only clears on user action.

Root causes from Apr 1: STO +75% missed (max_price_pct=30% blocked it). NOM +26% missed (chg_pct=0.0 from Binance). ONT +47% only 1 alert (60-min cooldown). All three now fixed.

v3.1.1 Apr 1, 2026

Surge Detector — min_price_pct fix

  • FIX surge_min_price_pct lowered 0.5% → 0.1%
  • FIX Catches intraday spikes on coins that are net-flat on 24h (e.g. GUSDT +12% intraday, +0.24% 24h — was excluded)

Root cause: GUSDT spiked +12% Apr 1 05:00 UTC with 10-17× 5m volume. Surge detector excluded it because 24h price change was only +0.24%, below the 0.5% gate. 24h change is the wrong metric for intraday spikes — the 5m vol gate is the real filter. Lowering to 0.1% only excludes near-zero stablecoins.
