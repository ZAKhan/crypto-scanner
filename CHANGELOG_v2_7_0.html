market_context() — cs/indicators.py
New function — 6-component structure analysis on last 25+ candles
EMA-50 slope as trend proxyLogic
Computes the percentage change in EMA-50 over the last 20 bars. Slope > +0.3% → trend "up" (+1). Slope < −0.3% → trend "down" (−1). Neutral slopes leave score unchanged. Requires at least 50 candles of data for the EMA to be meaningful.
Lower highs — 3 consecutive swing highs decliningLogic
Identifies swing highs as bars whose high exceeds both neighbours. If the last 3 swing highs form a descending sequence, lower_highs = True and score −2. This is the strongest bearish structural signal — classic lower-high downtrend.
Lower lows — 2 consecutive swing lows decliningLogic
Same approach applied to swing lows (bars whose low undercuts both neighbours). Two declining lows → lower_lows = True, score −1. Combined with lower highs, this fully confirms a bear structure.
Dump detector — big red candle with heavy volume in last 10 barsLogic
Scans the last 10 candles for a red bar with body > 0.5% of open price AND volume > 2× the candle-set average. Records the highest dump_vol_ratio found. Score −1. The dump's volume multiple is used in the next check.
Bounce quality — last 3 bars vs dump candle volumeLogic
Only runs if a dump was detected. Compares the average volume of the last 3 candles against the dump candle's volume. If bounce volume ≥ 60% of dump volume → real demand, score +1. Below that threshold → dead-cat bounce risk, score −1, bounce_vol_weak = True.
Volume character — last 5 bars buy vs sell pressureLogic
Splits the last 5 bars into buy volume (green candles) and sell volume (red candles). Buy ratio > 65% → "buying" (+1). Buy ratio < 35% → "selling" (−1). Shown in block_reason as sell pressure.

Signal Gate — cs/scanner.py
Downgrades BUY / STRONG BUY / PRE-BREAKOUT when structure_score ≤ −2
market_context() called after every positive signalNew
In Scanner._scan(), immediately after analyse() returns a BUY, STRONG BUY, or PRE-BREAKOUT signal, market_context(data["candles"]) is called on the already-fetched candle list — zero extra API calls. If structure_score <= −2, signal is overwritten to NEUTRAL and ctx_blocked = True. Otherwise ctx_blocked = False and the score is attached as ctx_score. The ctx_reason string is set in both cases.
Threshold: score ≤ −2, not −1Logic
A single bearish condition (e.g. EMA slope slightly negative) scores −1 and is not blocked — the signal still fires. The gate requires two or more concurrent bearish conditions before it acts. This avoids false negatives on coins that are just consolidating.

Signal Tooltip — cs/main_window.py
Hover the Signal column to see why a signal was or wasn't blocked
ctx_reason shown as tooltip on the Signal cellNew
When a result has a non-empty ctx_reason, the signal cell in the Scanner table now shows a tooltip. Blocked signals display "Blocked: lower-highs | dead-cat risk". Signals that passed show "Context: downtrend" so you can see the structural note even when the gate didn't fire. No new column — the information lives in the hover.
✓
Everything Else Unchanged
All existing signals, alerts, trades, and settings unaffected
SELL / STRONG SELL signals bypass the gate entirelyLogic
The market structure gate only runs on BUY-direction signals. SELL signals are not affected — bearish structure is expected context for a short signal, not a reason to block it.
Settings, signal log, trades, and alert engine unchanged
No config keys added. No CSV columns added. No alert engine changes. Settings files from v2.6.x load without migration. The gate runs silently inside the scan thread and writes only ctx_blocked, ctx_reason, and ctx_score keys onto the result dict.
