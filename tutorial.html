1 — Installation

1
Install Python dependencies
pip install PyQt6 requests websocket-client
On Arch/CachyOS: pip install PyQt6 requests websocket-client --break-system-packages
2
Run the app
python crypto_scanner.py
3
Or build a standalone binary
./build.sh
Installs to ~/.local/bin/crypto_scanner and registers in app menu under Finance.
💡
No API key needed for scanning. The scanner uses Binance public endpoints. API keys are only required for placing live trades.

Closing a Trade

Option 1 — Auto-close (recommended)
Set TP when opening. WebSocket detects price crossing TP → app places market SELL automatically within milliseconds.
Option 2 — Manual close
Right-click trade row → Close → confirm at current price. App cancels OCO and places market SELL.

OCO Stop-Loss Protection

⚡ Always protected — even if app closes

Every BUY automatically places an OCO order on Binance: a limit sell at TP and a stop-loss at SL.
When one fires, the other cancels automatically.

Your stop-loss lives on Binance's servers — it fires even if the app crashes, your PC restarts, or internet drops.

Trade Statistics

The stats panel shows: total trades, open, wins, losses, win rate, avg win/loss, best/worst trade, profit factor, total P&L, and equity curve chart.

9 — PRE-BREAKOUT Signal NEW v2.2

Early warning

PRE-BREAKOUT fires before a coin moves — not after. It detects coins quietly coiling before a price spike by requiring all 4 conditions simultaneously:

BB Width < 5%
Bollinger Bands are squeezed tight — price has been ranging in a very narrow band. Like a spring compressing before release.
Volume ≥ 1.5× average
Unusual volume building quietly. Smart money accumulating before the move.
RSI between 35-55
Not overbought, recovering from oversold. Price has room to move up.
Price in bottom 25% of BB range
Price sitting near the lower Bollinger Band — at support. Primed for a bounce.
💡
PRE-BREAKOUT appears first in both Scanner and Top Picks, above STRONG BUY. It has its own orange badge and distinct 3-tone ascending alert sound. Use it to get ahead of the move.

11 — Signal Audit Log NEW v2.2

Analysis & improvement

Every scan writes a full CSV log with all coins and their indicator values. Use it to analyse missed moves and improve your settings over time.

What is logged

22 columns per coin per scan: timestamp, symbol, price, change_24h, signal, confidence, rsi, stoch_rsi, macd_hist, bb_pct, bb_width_pct, vol_ratio, vol_spike, adr_pct, pattern, long_score, short_score, potential, exp_move, trend_1h, alert_fired, safety_blocked, safety_reason.

The most useful columns for analysis:

• alert_fired — did the alert system fire for this coin?
• safety_blocked — was a trade blocked by a safety rule?
• safety_reason — exactly which rule blocked it and why

File location

OSPath
Linux~/.config/CryptoScalper/logs/signal_log_YYYY-MM-DD.csv
Windows%APPDATA%\CryptoScalper\logs\signal_log_YYYY-MM-DD.csv
macOS~/Library/Application Support/CryptoScalper/logs/signal_log_YYYY-MM-DD.csv

A new file is created each day. Files older than 7 days are deleted automatically.

How to open it

Config tab → Export Scan Results → 📋 Open Signal Log. Opens today's file in LibreOffice Calc or Excel. The size label shows current row count and total size of all log files.

How to use it for improvement

1
Find a missed move
e.g. CFG spiked 13% at 11am — you didn't get an alert
2
Open the log and filter by symbol + time
Look at CFG rows from 10:00-11:00. What were the RSI, BB%, volume ratio, ADR values?
3
Check alert_fired and safety_blocked columns
If alert_fired=False — which filter was too strict? If safety_blocked=True — check safety_reason.
4
Adjust thresholds accordingly
Lower Min ADR%, raise Max RSI, or adjust BB% limit. Save settings and watch future scans.

12 — How Scoring Works

The signal engine

Every coin gets scored independently for LONG and SHORT. The higher-scoring side wins provided it clears a minimum margin. Pure deterministic rules — no machine learning, no black box.

IndicatorLONG pointsSHORT pointsMax
RSI <25→+5 · <30→+4 · <35→+3 · <40→+2 · <45→+1 >75→+5 · >70→+4 · >65→+3 · >60→+2 · >55→+1 ±5
Stoch RSI <20→+2 · <40→+1 >80→+2 · >60→+1 ±2
MACD Positive + rising→+3 · Positive stale→+1 Negative + falling→+3 · Negative stale→+1 ±3
Bollinger Band pos<10%→+3 · <25%→+2 · <40%→+1 pos>90%→+3 · >75%→+2 · >60%→+1 ±3
Pattern Bullish pattern→+2 · Bearish→−1 Bearish pattern→+2 · Bullish→−1 ±2
24h Momentum 24h<−10%→+1 (bounce candidate) 24h>15%→+2 · >8%→+1 ±2
Signal thresholds:

STRONG BUY — Long score ≥ 6 AND margin over Short ≥ 3
BUY — Long score ≥ 3 AND margin ≥ 2
PRE-BREAKOUT — BB squeeze + volume + RSI 35-55 + price at support (separate logic)
STRONG SELL — Short score ≥ 6 AND margin over Long ≥ 3
SELL — Short score ≥ 3 AND margin ≥ 2
NEUTRAL — Neither side clears the threshold

13 — Full Workflow Example

End-to-end scalp trade
1
Configure filters (Config tab)
Set Max Price $1, Min Volume $1M, SL% 3, TP% 5. Review Trade Safety rules. Hit Apply Settings.
2
Enable auto-scan (Alerts tab)
Check "Enable auto-scan alerts", set interval 30s, Minimum signal = STRONG BUY, Min ADR = 0.5%. Apply Alert Settings.
3
App scans automatically — WebSocket connects
After first scan, ⚡ WS turns green in status bar. Prices now update in real-time.
4
Alert fires for STRONG BUY on COS
Sound plays + desktop notification. Check Top Picks tab — COS appears in LONG CANDIDATES.
5
Safety rules pass (signal held 2 scans, BTC not falling)
Right-click COS → BUY → dialog opens with pre-filled entry, SL, TP.
6
Confirm trade → BUY executes on Binance
OCO placed automatically. Trade appears in Trades tab with live P&L updating via WebSocket.
7
TP hit → auto-closes in milliseconds
WebSocket detects price crossing TP. Market SELL placed, OCO cancelled. Trade shows as WIN.
8
Review signal log for analysis
Config → Export → 📋 Open Signal Log. Check what the indicators looked like before the move.
