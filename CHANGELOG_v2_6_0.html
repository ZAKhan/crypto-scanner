Modular Package — cs/
Single-file monolith → 15 focused modules
crypto_scanner.py becomes a thin entry pointRefactor
The 8,214-line monolith has been split into a cs/ Python package. crypto_scanner.py now contains only 5 lines — it imports and calls main() from cs.main_window. All logic lives in the package. Run command, build script, and binary output are unchanged.
Each module owns one concern — edit only what you needImproved
Touching indicator logic? Edit cs/indicators.py. Tweaking the stylesheet? cs/stylesheet.py. Adding a safety rule? cs/safety.py. No more scrolling through 8,000 lines to find a function. Cross-module imports use explicit from cs.module import name paths so dependencies are always visible.

build.sh — added --collect-all csImproved
PyInstaller now explicitly bundles the entire cs/ package via --collect-all cs. Entry point and output path are unchanged — binary is still produced at dist/linux/crypto_scanner.
push_release.sh — stages cs/ package, reads version from cs/config.pyImproved
APP_VERSION is now read from cs/config.py (previously crypto_scanner.py). The entire cs/ directory is staged automatically. cleanup.sh added to core files. Changelog reference updated to CHANGELOG_v2_6_0.html.
cleanup.sh — cs/ directory added to keep listImproved
The cs/ package directory is now in the KEEP list so cleanup.sh never moves it to backup.
